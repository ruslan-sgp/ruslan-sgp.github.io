Data Analysis & Modeling Concepts

1. Measures of Central Tendency and Variation

  • Central Tendency: A single value that describes the center of a dataset.
    • Mean: The average (sum of all values / number of values).
    • Median: The middle value when data is sorted.
    • Mode: The most frequently occurring value.
  • Variation (Dispersion): Describes how spread out the data is.
    • Range: Max value - Min value.
    • Variance: The average of the squared differences from the Mean.
    • Standard Deviation: The square root of the Variance. The most common measure of spread.

2. Correlation

  • Measures the strength and direction of the linear relationship between two variables.
  • Value ranges from -1 to +1.
    • +1: Perfect positive correlation (as one goes up, the other goes up).
    • -1: Perfect negative correlation (as one goes up, the other goes down).
    • 0: No linear correlation.
  • Note: Correlation does not imply causation.

3. Regression

  • A technique to model the relationship between a dependent variable (Y) and one or more independent variables (X).
  • The goal is to find a function (a line or curve) that best predicts Y from X.
  • Simple Linear Regression: Y = β₀ + β₁*X (a straight line).

4. One-Factor vs. Multi-Factor Models

  • One-Factor (Univariate) Model: A model that uses only one independent variable (feature) to predict an outcome.
    • Example: Predicting house price based only on its size.
  • Multi-Factor (Multivariate) Model: A model that uses multiple independent variables to predict an outcome.
    • Example: Predicting house price based on size, number of bedrooms, age, and location.

5. Error Metrics (How good is the model?)

Used to evaluate the difference between a model's predictions and the actual values.

  • MAE (Mean Absolute Error): The average of the absolute errors. Easy to understand.
  • MSE (Mean Squared Error): The average of the squared errors. Punishes large errors more than MAE.
  • RMSE (Root Mean Squared Error): The square root of MSE. Punishes large errors and is in the same units as the target variable (often more interpretable than MSE).

6. Accuracy

  • Primarily used for classification models (predicting categories, e.g., spam/not spam).
  • Defined as: (Number of Correct Predictions) / (Total Predictions)

7. Train & Test Split

  • A fundamental best practice to evaluate a model's performance fairly.
  • Training Set: The portion of the data (e.g., 70-80%) used to build (train) the model.
  • Test Set: The withheld portion of the data (e.g., 20-30%) used to evaluate the model's performance on unseen data. This tests how well the model generalizes.

8. Fit & Predict

  • fit(X_train, y_train): The core training process. The algorithm learns the parameters (e.g., the slope β₁ and intercept β₀ of a line) from the training data.
  • predict(X_test): Using the learned model/parameters to make predictions on new data (e.g., the test set).
  • Parameters: These are the model's coefficients learned during the fit() process (e.g., the weights in a regression equation). They are internal to the model.
